# 2026-06-09 자바 학습 정리 : 생성자

## 오늘 배운 내용

- **기본형(primitive)** , **참조형(reference)**
- **생성자(constructor)**

## 기억할 것

- Java 에서는 int, double, float, bool, char 등 값을 저장하는 기본형 (primitive type) C에서는 일반 변수
- Hero 와 같이 new 로 생성되는 인스턴스인 “레퍼런스 타입" String 은 레퍼런스 타입이다.
- Java 와 다르게 C++에서는 클래스 타입도 일반변수로 쓸 수 있다.
- C++ 의 래퍼런스 변수는 선언과 동시에 상수처럼 대입한다면 Java는 선언까지는 할 수 있고 나중에 대입하여 사용이 가능하다.
- Java의 **래퍼런스 타입**은 결국 **포인터 변수**와 비슷한 역할이다.

## 실습 코드

```java
package com.survivalcoding;

import org.junit.jupiter.api.BeforeEach;
import org.junit.jupiter.api.Test;

import static org.junit.jupiter.api.Assertions.*;

class ClericTest {
    Cleric cleric;

    @BeforeEach
    void setUp() {
        // Given: 테스트 시작 전 항상 새로운 Cleric 객체 생성
        cleric = new Cleric("아서스");
    }

    @Test
    void constructor_Test() {
        // Given + When : 이름 아서스 , HP 40 , MP 5
        Cleric c1 = new Cleric("아서스", 40, 5);

        // Then : 아서스 , 40 , 5 검증
        assertEquals("아서스", c1.name);
        assertEquals(40, c1.HP);
        assertEquals(5, c1.MP);

        // Given + When : 이름 아서스 , HP 35 , MaxMP 10
        Cleric c2 = new Cleric("아서스", 35);

        // Then : 아서스 , 35 , 10 검증
        assertEquals("아서스", c2.name);
        assertEquals(35, c2.HP);
        assertEquals(10, c2.MP);

        // Given + When : 이름 아서스 , MaxHP 50 , MaxMP 10 ( 이미 위에서 @BeforeEach로 생성된 인스턴스 활용 )

        // Then : 아서스 , 50 , 10 검증
        assertEquals("아서스", cleric.name);
        assertEquals(50, cleric.HP);
        assertEquals(10, cleric.MP);
    }
}
```

## 어려웠던 점

- 자바의 참조형(Reference) 변수와 C++의 포인터/레퍼런스 개념이 뒤섞여 메모리 구조를 이해할 때 혼란스러웠다.
- 생성자를 테스트할 때는 기존 기능 테스트와 달리 Given-When-Then 패턴의 경계가 모호해서 어색했다.

## 해결 방법

- 자바의 참조 변수는 결국 내부적으로 **C++의 포인터 변수와 99% 똑같이 주소값을 다루는 상자**라는 본질을 꿰뚫어 보며 의문이 해결되었다. (단지 주소 연산만 못 하도록 안전하게 포장된 것뿐!)
- 생성자 테스트는 객체가 '태어나는 것' 자체가 목적이므로, 객체를 생성하는 행위가 준비(Given)이자 실행(When)이 되어 한 줄로 묶이는 것이 당연함을 깨달았다.
