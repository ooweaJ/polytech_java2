# 2026-06-29 자바 학습 정리 : 제네릭, 열거형

## 오늘 배운 내용

- 제네릭 (Generic)
- 열거형 (enum)

## 기억할 것

- Java의 제네릭(Generic)은 C++의 템플릿(Template)과 구조 및 철학이 완전히 같다.
    - StrongBox< E > 처럼 타입을 미정으로 두고 객체 생성 시 주입하는 방식이 동일하다.
    - 덕분에 데이터를 꺼낼 때 별도의 타입 캐스팅(강제 형변환)을 안 해도 원래 타입 그대로 안전하게 반환받는다.


- Java의 enum은 C++과 달리 내부적으로 인스턴스가 자동 생성되는 '클래스'다.
    - enum 클래스 하나를 선언하면 내부에 적어준 상수 개수만큼 인스턴스가 생성된다. (예제에서는 총 4개의 열쇠 인스턴스 생성)

## 실습 코드 (테스트 구조 검증)

```Java
public class StrongBoxTest {

    @Test
    @DisplayName("PADLOCK 열쇠로 금고를 1024번 호출하여 내용물이 정상적으로 반환 그 전까지 NULL 반환")
    void PADLOCK_Test() {
        StrongBox<String> box = new StrongBox<String>(KeyType.PADLOCK);
        box.put("골드바");

        for (int i = 1; i < 1024; i++) {
            assertNull(box.get(), "1024 미만 일 때는 Null을 반환합니다");
        }

        assertEquals("골드바", box.get(), "1024번째에는 데이터가 정상 출력되어야 합니다");
    }

    @Test
    @DisplayName("BUTTON 열쇠로 금고를 10000번 호출하여 내용물이 정상적으로 반환 그 전까지 NULL 반환")
    void BUTTON_Test() {
        StrongBox<String> box = new StrongBox<String>(KeyType.BUTTON);
        box.put("비트코인");

        for (int i = 1; i < 10000; i++) {
            assertNull(box.get(), "10000 미만 일 때는 Null을 반환합니다");
        }

        assertEquals("비트코인", box.get(), "10000번째에는 데이터가 정상 출력되어야 합니다");
    }
}

``````

## 어려웠던 점

- Java의 enum에서 각 enum 상수 뒤의 괄호 값이 생성자의 매개변수랑 타입이 같다면 매개변수로 전달된다는 점

### ex)

```java
enum KeyType {
    PADLOCK(1024),
    BUTTON(10000),
    DIAL(30000),
    FINGER(1000000);

    private final int limit;

    KeyType(int limit) {
        this.limit = limit;
    }
}
```

Java에서 이런 문법이 가능한 핵심 이유는 enum 상수 하나하나가 해당 enum 타입의 인스턴스이기 때문이서였다.

## 해결 방법

- Java의 enum은 클래스이고 초기화가 될 때 모든 enum 인스턴스가 생성된다.
