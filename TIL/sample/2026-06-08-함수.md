# 2026-06-08 함수

## 오늘 배운 내용

- 함수는 여러 줄의 코드를 하나의 이름으로 묶어 재사용할 수 있게 해준다.
- Java에서는 클래스 안에 함수를 작성하며, 보통 메서드라고 부른다.
- 함수는 입력값을 받을 수 있고, 실행 결과를 반환할 수 있다.
- 같은 동작을 여러 번 작성하지 않고 함수로 분리하면 코드가 더 읽기 쉬워진다.

## 기억할 것

- 함수 이름은 동작을 잘 나타내도록 작성하는 것이 좋다.
- 매개변수는 함수가 실행될 때 외부에서 받는 값이다.
- 반환값이 없는 함수는 `void`를 사용한다.
- 반환값이 있는 함수는 반환 타입을 적고, `return`으로 값을 돌려준다.
- 함수는 한 가지 역할에 집중하도록 만드는 것이 좋다.

## 실습 코드

```java
public class Main {
    public static void main(String[] args) {
        printWelcomeMessage();

        int result = add(10, 20);
        System.out.println("10 + 20 = " + result);

        boolean adult = isAdult(20);
        System.out.println("성인 여부: " + adult);
    }

    static void printWelcomeMessage() {
        System.out.println("Java 함수 연습");
    }

    static int add(int a, int b) {
        return a + b;
    }

    static boolean isAdult(int age) {
        return age >= 19;
    }
}
```

## 어려웠던 점

- `void`를 언제 쓰는지 헷갈렸다.
- 매개변수와 반환값의 차이가 처음에는 명확하지 않았다.
- `return`을 쓰면 함수 실행이 끝난다는 점을 놓쳤다.

## 해결 방법

- 결과를 돌려줄 필요가 없으면 `void`를 사용한다고 정리했다.
- 함수 안으로 들어오는 값은 매개변수, 함수 밖으로 나가는 값은 반환값이라고 생각했다.
- `return` 아래에 코드를 작성하면 실행되지 않을 수 있다는 점을 예제로 확인했다.
