# 2026-06-08 자바 학습 정리: 객체지향과 테스트

## 오늘 배운 내용

- **클래스(Class):** 객체를 만들기 위한 '설계도' 또는 '틀'입니다. (예: `Cleric` 클래스)
- **객체(Object):** 클래스를 기반으로 메모리에 생성된 실체입니다.
- **인스턴스(Instance):** 특정 클래스로부터 생성된 개별적인 객체를 의미합니다.
- **메서드(Method):** 클래스 내부에 정의된 함수로, 객체의 동작(기능)을 수행합니다.
- **함수 vs 메서드:** 일반적으로 혼용하지만, 자바에서는 클래스 내부에 포함된 함수를 메서드라고 부릅니다.
- **테스트(Test):** 작성한 코드가 의도대로 동작하는지 확인하는 과정입니다.
- **JUnit 5:** 자바에서 테스트를 작성하기 위한 표준 라이브러리입니다.
    - `@Test`: 테스트 메서드임을 명시합니다.
    - `@BeforeEach`: 각 테스트가 실행되기 직전에 실행되어 테스트 환경을 초기화합니다.

## 기억할 것

- **객체지향 설계:** 데이터(상태)와 기능(메서드)을 하나의 클래스에 모아서 관리한다.
- **메서드 작성:** 객체의 상태를 변경하거나 값을 계산하는 역할을 명확히 한다.
- **테스트의 중요성:** 코드를 작성한 뒤 "정말 잘 동작하는지" 확인하는 것이 프로그래밍의 완성이다.
- **Given-When-Then:** 테스트 코드를 작성할 때 [상황 설정 - 실행 - 검증]의 흐름을 지키면 가독성이 좋아진다.

## 실습 코드 (테스트 코드 예시)

```java
package com.survivalcoding;

import org.junit.jupiter.api.BeforeEach;
import org.junit.jupiter.api.Test;

import static org.junit.jupiter.api.Assertions.*;

class ClericTest {
    Cleric cleric;

    @BeforeEach
    void setUp() {
        // Given: 테스트 시작 전 항상 새로운 Cleric 객체 생성
        cleric = new Cleric();
    }

    @Test
    void selfAid_WhenHpIsLow_ThenHpBecomesMaxAndMpDecreases() {
        // Given
        cleric.HP = 20;
        cleric.MP = 10;

        // When
        cleric.selfAid();

        // Then
        assertEquals(50, cleric.HP);
        assertEquals(5, cleric.MP);
    }
}
```

## 어려웠던 점

- 클래스와 인스턴스의 개념이 추상적이어서 처음엔 혼란스러웠다.
- 테스트 코드를 왜 따로 작성해야 하는지 이해가 필요했다.
- Given-When-Then 패턴을 코드에 적용하는 과정이 조금 어색했다.

## 해결 방법

- 클래스는 '붕어빵 틀', 객체/인스턴스는 '찍어낸 붕어빵'으로 이해했다.
- 테스트 코드는 코드가 바뀔 때마다 사람이 일일이 실행해보지 않아도 자동으로 안정성을 검증해주기 때문에 필수적임을 깨달았다.
- 반복적으로 테스트 코드를 작성하며 Given-When-Then 구조에 익숙해졌다.
